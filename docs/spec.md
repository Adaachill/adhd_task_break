# 製品設計書：『タスクブレーキ（仮称）』

## 0. 製品コンセプト要約

**「LINEの手軽さで吐き出し、松竹梅で着手し、強制アラートで過集中を止める、ADHD特化型行動コントロールアプリ」**

ADHD当事者の「行動のハードル（やる気ゼロ）」と「過集中（やりすぎ）」を *同時に* 解決する
ネイティブタスク管理アプリ。世のタスク管理が「サボり防止」に偏る中、「やりすぎてしまう」という
ADHD特有の痛みにスポットを当てる。

- **ターゲット：** タスク管理が続かない／メモが溜まり優先順位を見失う／着手に莫大な時間がかかる／
  一度集中すると時間を忘れて数時間やり続けてしまう（過集中・タスクすり替え）ADHD傾向の方。
- **配信形態：** ネイティブアプリ（React Native + Expo）。バックグラウンドの強制アラート・
  常時バナーが体験の核であり、iOS PWA では実現不可のためネイティブを採用。

## 1. 機能検証（最小性・訴求の検証）

### 1.1 最小性レビュー（MVP要否判定）

| 機能 | MVP判定 | 根拠 |
|---|---|---|
| ① 脳内吐き出し（チャットUI入力） | **必須** | 全エントリポイント。摩擦ゼロ入力が体験の核 |
| ② 松竹梅（梅クリア） | **必須** | 差別化の核・成功体験の源。訴求軸そのもの |
| ③ 強制ブレーキタイマー | **必須** | 「やりすぎ防止」の唯一性。ネイティブ採用の理由 |
| ④-a 時間計測 | **必須** | ⑤の褒めログに必要 |
| ④-b 単発/習慣切替 | **MVP導入** | 継続タスクの可視化に必要 |
| ⑤ 褒めログ（未完了非表示） | **必須** | 継続率・自己嫌悪防止。リテンションの核 |
| ⑥ 作業中離脱抑止（バナー＋ポップアップ） | **MVP導入** | 過集中対策の補強。離脱→引き戻しでメタ認知を促す |
| AI自動分類 | **MVP導入** | テキストから🔵/🔥・期限を自動判定。手動1タップは補正手段として併存 |

**MVPスコープ確定：① ② ③ ④-a ④-b ⑤ ⑥ ＋ AI自動分類**

### 1.2 コンセプト上の論点と対応

1. **🔵/🔥 は排他属性にする** — 1タスク1属性で単純化。🔵には松竹梅、🔥にはブレーキタイマーを
   割り当て、UIを迷わせない。
2. **「今日の3タスク」への昇格動線** — ADHDユーザーは優先順位付けが苦手。吐き出しリストから
   カードを1タップ/スワイプで「今日やる」に昇格させる低摩擦動線を用意。3枠上限で選択麻痺を防ぐ。
3. **タイマーは絶対時刻ベース** — 経過秒のカウントではなく開始タイムスタンプ＋OSスケジュール通知で
   実装し、バックグラウンド/アプリ kill でもブレーキが発火するようにする。
4. **オンボーディング不要** — 起動直後に吐き出し可能にする（摩擦最小）。
5. **離脱検知の技術的事実** — モバイルOSのサンドボックス制約上、アプリは「ユーザーが他アプリを
   開いたこと」自体を検知できない。代替として *自アプリがバックグラウンド化した（AppState→background）*
   をトリガにし、作業セッション中であれば「〇〇の作業中です。関連ありますか？」のローカル通知/
   ポップアップを発火、常時バナーは Android＝フォアグラウンドサービスの常駐通知 /
   iOS＝Live Activity で実現する。
6. **通知強度はユーザー選択式** — 設定で「マナーモード貫通（強）」/「通常バイブ＋音（標準）」を切替。
   MVPは標準のみ実装し、貫通モードは設定UIを置きつつ段階有効化。

### 1.3 訴求検証（響く構成か）

訴求3軸はいずれも当事者の痛みに直結し共感性が高く、構成は妥当。
- **「やる気ゼロ」と「やりすぎ」を同時に解決する唯一のアプリ**
- **「意識の低さ」を全肯定する「梅クリア」システム**（例：「教科書を1ページ開いたから今日はクリア！」）
- **LINEのゴミ箱化からの救済**（自分用グループが未完了タスクで溢れて病む課題への解決ストーリー）

補強：③のブレーキ警告画面・⑤の褒めログをスクショ映えする演出にし、X等での拡散導線を画面設計に組み込む。

## 2. 画面イメージ（ワイヤーフレーム）

### 画面1：脳内吐き出し（チャット）

- LINE風チャットバブル。新しいものが下。
- 送信時に **AIが🔵/🔥・期限を自動付与**（バッジ表示）。バッジタップで1タップ補正。
- 各タスクに「今日やる↑」昇格ボタン（3枠まで）／単発・🔁習慣トグル。

### 画面2：今日の3タスク（実行）

- 単発 / 🔁習慣 タブをワンタップ切替。最大3枚、空枠は「+ 吐き出しから選ぶ」。
- 🔵：松竹梅3ボタン、梅達成で即「できた！」演出。
- 🔥：制限時間プルダウン（既定45分・必須）→「ブレーキ開始」。
- 実行中は上部に常時バナー「〇〇を作業中」。

### 画面3：本日の褒めログ（振り返り）

- 大きく「今日 〇〇分 動けた！」。
- クリアしたタスクのみ列挙＋梅/竹/松バッジ。未完了は非表示（自己嫌悪防止）。
- 「結果をシェア」ボタン（SNS拡散）。

### オーバーレイA：ブレーキ警告（タイムアップ時）

- 通知強度はユーザー設定（貫通/標準）に従う。
- 「⏱ 15分延長（ペナルティ表示）」/「✋ 強制終了して休憩」の2択のみ。
- スクショ映え演出で拡散導線。

### オーバーレイB：離脱抑止（作業中に自アプリがバックグラウンド化）

- ローカル通知/復帰時ポップアップで引き戻す。
- 作業セッション中は常時バナーを表示し続ける。
- 技術的注記：他アプリ起動の直接検知は不可。AppStateのbackground遷移をトリガにする。

## 3. 技術構成（実走アーキテクチャ）

### 3.1 フロント（ネイティブ）

- **React Native + Expo（TypeScript）** + EAS Build。
- 状態管理：Zustand もしくは hooks＋ローカルDB。
- **オフラインファースト**：タスクデータはローカル完結。永続化に **expo-sqlite**（または WatermelonDB）。
  ADHDのタスクは機微情報のためローカル完結はプライバシー上も有利。

### 3.2 通知・ブレーキの実装（最重要・リスク明記）

- スケジュール通知：`expo-notifications`（ローカル通知はアプリkill時も発火）。
- 振動：`expo-haptics` / Vibration。
- **通知強度2モードを設定で選択**：
  - 「標準（通常バイブ＋音）」＝MVP必須。`expo-notifications` の通常通知＋`expo-haptics`。
  - 「マナーモード貫通（強）」＝OS制約ありのため段階対応：
    - iOS：**Critical Alerts** はApple特別エンタイトルメント審査が必要。未承認時は
      **Time-Sensitive通知**がフォールバック。
    - Android：**フルスクリーンIntent＋フォアグラウンドサービス**でアラーム的挙動を実現。
    - 高度な制御が必要なら **Notifee**（フルスクリーンIntent/critical対応）採用を検討。
      Expo managedでは config plugin / dev client が必要になりうる。
- **作業中の常時バナー＆離脱抑止**：
  - Android＝フォアグラウンドサービスの常駐通知、iOS＝**Live Activity**
    （Expo対応プラグイン or Notifee/カスタムネイティブモジュール）。
  - 離脱検知は `AppState` の `background` 遷移をトリガに、作業セッション中ならポップアップ用
    ローカル通知を発火。**他アプリ起動の直接検知はOS制約で不可**。
- タイマーは絶対時刻ベース（開始タイムスタンプ保存）でバックグラウンド誤差を排除。

### 3.3 AI自動分類（MVP導入）

- テキスト→🔵/🔥・期限(今日/明日/いつか)を判定。
- 構成案：軽量バックエンド（FastAPI）に分類エンドポイントを置き、Claude API 等のLLMで判定。
  プロンプトキャッシュでコスト最適化。
- フォールバック：オフライン時/低信頼時は未分類＋手動1タップ補正で必ず成立させる。
- プライバシー：吐き出しテキストを外部送信するため、設定で「AI分類オフ（手動のみ）」を選べる。

### 3.4 バックエンド

- タスクデータはローカル完結（**expo-sqlite**）でオフラインファースト。
- AI分類のため軽量 **FastAPI** を導入。分類のみのステートレスAPIから開始。
- クラウド同期/アカウントは v2（認証＋PostgreSQL）。

### 3.5 段階リリース計画

- **v1（MVP）**：①〜⑥全機能＋AI自動分類・習慣切替・通常バイブ通知・離脱抑止バナー。
  ローカル完結＋分類API。
- **v2**：マナーモード貫通アラートの本実装（iOS Critical Alerts審査）／クラウド同期・アカウント／
  シェア演出強化。
